Nav

	
			<ul class="nav nav-tabs">
	  		<li role="navigation" class="active"><a href="#">Home</a></li>
	  		<li role="navigation"><a href="#">Profile</a></li>	 
	  		<li role="navigation"><a href="#">Messages</a></li>
			</ul>
  • Navs available in bootstrap have a shared markup, starting with the base .nav class, as well as shared states. Swap Modifier classes to swtich between each style
  • For tables with a clickable area, the tabs JavaScript plugin must be in use. The markup will aslo require additional role and ARIA attribues.
  • If navs are being used to provide a navigation bar, make sure to add role="navigation" to the most logical parent container of the <ul> or wrap a <nav> element around the whole navigation. Do not add the role to the <ul> itself, as this would prevent it from being announced as an actual list by assistive technologies.

Syle of buttons

  • Tabs .nav-tabs
    <ul class="nav nav-tabs">
  • Pills.nav-pills
    <ul class="nav nav-pills">
  • Pills Stacked.nav-pills nav-stacked. This will place the buttons verticly

Justified

  • Easily make tabs or pills equal qidths of their parent at screen wider than 768px with .nav-justified. on smaller screens, the nav links are stacked.
